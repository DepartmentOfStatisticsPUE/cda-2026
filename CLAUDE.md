# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Teaching repository for **Categorical Data Analysis (CDA) 2025/26** at Poznań University of Economics and Business. Course materials are authored as Quarto `.qmd` files, which are rendered to HTML and have code blocks automatically extracted into per-language scripts and Jupyter notebooks.

## Build Pipeline

Requires **Quarto >= 1.7.0** (needed by the Ripper extension) and Python 3 with `nbformat`. All build commands run from the `.tools/` directory:

```bash
cd .tools && make extract    # Full pipeline: render → sort → notebooks → readme
cd .tools && make publish    # extract + git add -A + commit + push
cd .tools && make render     # Incremental: re-render only .qmd files newer than their .html
cd .tools && make render-all # Force re-render every .qmd (use when filters/templates change)
cd .tools && make sort       # Only move extracted files to codes/{R,python,julia}/
cd .tools && make notebooks  # Only generate Jupyter notebooks from .py files
cd .tools && make readme     # Only update readme.md download table
cd .tools && make clean      # Remove generated files (preserves HTML)
```

Single-file notebook generation: `cd .tools && python3 py_to_notebook.py 05-mle.qmd`

**After any `.qmd` edit, always run `make extract`** to keep generated files in sync.

## Architecture

### Source of Truth

All content lives in `codes/qmd/*.qmd`. These files contain R, Python, and Julia code blocks plus markdown. **Never directly edit** generated files — they will be overwritten on the next `make extract`:

- `codes/R/`, `codes/python/`, `codes/julia/` — extracted language-specific scripts
- `codes/notebooks/*.ipynb` — Jupyter notebooks (Python only)
- `readme.md` download table — auto-generated by `update_readme.py`

### QMD File Structure

Each `.qmd` uses the `ripper` filter in its YAML frontmatter to enable code extraction. The setup chunk hardcodes local engine paths (JuliaCall, Python, Julia) — these are author-specific and should not be changed for portability.

Most topic sections use `::: panel-tabset` with `## R` / `## Python` / `## Julia` tabs to show equivalent code in all three languages. When editing a code example, update all three language tabs to keep them in sync. Not all topics have all three languages (e.g., `02-simpson-paradox` is R-only, `09-glm-lr` is currently Python-only).

### New QMD Frontmatter Template

```yaml
---
title: "Topic Title"
author: "Maciej Beręsewicz"
format: 
  html:
    embed-resources: true
    table-of-contents: true
    number-sections: true
editor: source
execute: 
  eval: true
  warning: false
  message: false
filters:
  - ripper
ripper:
  output-dir: "extracted"
  script-links-position: "none"
  include-yaml: true
---
```

### Pipeline Stages

1. `quarto render *.qmd` → HTML (with executed output) + Ripper Lua filter extracts `.R`, `.py`, `.jl` files
2. `make sort` → moves extracted files from `codes/qmd/` to `codes/{R,python,julia}/`
3. `py_to_notebook.py` → converts `.py` scripts + `.qmd` section headers → `.ipynb`
4. `update_readme.py` → regenerates download table in `readme.md` + `moodle-table.html` (the latter is gitignored; local convenience only)

### Key Components

- **Ripper** (`codes/qmd/_extensions/coatless-quarto/ripper/`): Lua filter that extracts code blocks during Quarto rendering — do not modify without understanding its extraction rules
- **`py_to_notebook.py`**: Parses `.qmd` for section headers and Python blocks to build notebooks with proper cell structure
- **`update_readme.py`**: Scans output directories and generates GitHub raw-link tables. Contains hardcoded `TOPIC_ORDER` and `TOPIC_NAMES` dicts that must be updated when adding new topics.

### Adding a New Topic

1. Create `codes/qmd/XX-topic.qmd` using the frontmatter template above
2. Add the topic stem to `TOPIC_ORDER` list and `TOPIC_NAMES` dict in `.tools/update_readme.py`
3. Add the notebook link to the "Outline of lecture" section in `readme.md` (this section is manually maintained — only the download table is auto-generated by `make readme`)
4. Run `cd .tools && make extract`

## Slides

Lecture slides are LaTeX Beamer in `slides/cda2026/cda.tex` (single monolithic file, all lectures).

```bash
cd slides/cda2026 && pdflatex cda.tex && bibtex cda && pdflatex cda.tex && pdflatex cda.tex
```

For quick iteration without bibliography changes, a single `pdflatex cda.tex` suffices. The full sequence is needed when citations change (`settings/bibliography.bib`).

The `lectures/` directory holds supplementary figures referenced by QMD notebooks.

## Homework & Grading

```bash
cd homeworks && Rscript generate-all-keys.R          # Regenerate all grading keys
cd homeworks && Rscript hw01-grading-key.R <student_id>  # Grade one student
```

Grading keys and solutions are excluded from git (see `.gitignore`): anything matching `homeworks/*-grading-key*`, `*-answer-key*`, `*-solution-*`, `generate-all-keys*`. Student submissions are `.qmd`/`.ipynb`/`.html`.

The `zaliczenie/` directory (private gradebooks, `cda-dzienne-*.csv` / `cda-zaoczne-*.csv`) is fully gitignored. The `example-test/` directory holds the example final exam (`example-test.qmd` → html), which is public.

## Data

Single main dataset: `data/polish-jvs.csv` — Polish job vacancy survey (~57K rows). Used across all topic notebooks.

## Packages

See the "Required packages / modules" section in `readme.md` for the canonical list of R, Python, and Julia packages used across notebooks. Use these when writing new code blocks.

## Distribution

Students access materials via `htmlpreview.github.io` links for HTML notebooks, and raw GitHub links for code downloads. `make publish` handles the full commit+push cycle.

In `readme.md`, the "Outline of lecture" section (with htmlpreview links) is manually maintained. Only the "Code files for download" table is auto-generated by `make readme`.
