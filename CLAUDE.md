# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Teaching repository for **Categorical Data Analysis (CDA) 2025/26** at Poznań University of Economics. Course materials are authored as Quarto `.qmd` files, which are rendered to HTML and have code blocks automatically extracted into per-language scripts and Jupyter notebooks.

## Build Pipeline

All build commands run from the `.tools/` directory:

```bash
cd .tools && make extract    # Full pipeline: render → sort → notebooks → readme
cd .tools && make publish    # extract + git add -A + commit + push
cd .tools && make render     # Only quarto render (no-execute)
cd .tools && make sort       # Only move extracted files to codes/{R,python,julia}/
cd .tools && make notebooks  # Only generate Jupyter notebooks from .py files
cd .tools && make readme     # Only update readme.md download table
cd .tools && make clean      # Remove generated files (preserves HTML)
```

Single-file notebook generation: `cd .tools && python3 py_to_notebook.py 05-mle.qmd`

**After any `.qmd` edit, always run `make extract`** to keep generated files in sync.

## Architecture

### Source of Truth

All content lives in `codes/qmd/*.qmd`. These files contain R, Python, and Julia code blocks plus markdown. **Never directly edit** generated files — they will be overwritten on the next `make extract`:

- `codes/R/`, `codes/python/`, `codes/julia/` — extracted language-specific scripts
- `codes/notebooks/*.ipynb` — Jupyter notebooks (Python only)
- `readme.md` download table — auto-generated by `update_readme.py`

### QMD File Structure

Each `.qmd` uses the `ripper` filter in its YAML frontmatter to enable code extraction. The setup chunk hardcodes local engine paths (JuliaCall, Python, Julia) — these are author-specific and should not be changed for portability.

### Pipeline Stages

1. `quarto render *.qmd --no-execute` → HTML + Ripper Lua filter extracts `.R`, `.py`, `.jl` files
2. `make sort` → moves extracted files from `codes/qmd/` to `codes/{R,python,julia}/`
3. `py_to_notebook.py` → converts `.py` scripts + `.qmd` section headers → `.ipynb`
4. `update_readme.py` → regenerates download table in `readme.md` + `moodle-table.html`

### Key Components

- **Ripper** (`codes/qmd/_extensions/coatless-quarto/ripper/`): Lua filter that extracts code blocks during Quarto rendering — do not modify without understanding its extraction rules
- **`py_to_notebook.py`**: Parses `.qmd` for section headers and Python blocks to build notebooks with proper cell structure
- **`update_readme.py`**: Scans output directories and generates GitHub raw-link tables

## Slides

Lecture slides are LaTeX Beamer in `slides/cda2026/cda.tex` (single monolithic file, all lectures). Build with `pdflatex` or your preferred LaTeX toolchain from that directory.

## Homework & Grading

```bash
cd homeworks && Rscript generate-all-keys.R          # Regenerate all grading keys
cd homeworks && Rscript hw01-grading-key.R <student_id>  # Grade one student
```

Grading keys and solutions are excluded from git (see `.gitignore`). Student submissions are `.qmd`/`.ipynb`/`.html`.

## Data

Single main dataset: `data/polish-jvs.csv` — Polish job vacancy survey (~57K rows). Used across all topic notebooks.

## Distribution

Students access materials via `htmlpreview.github.io` links for HTML notebooks, and raw GitHub links for code downloads. `make publish` handles the full commit+push cycle.
